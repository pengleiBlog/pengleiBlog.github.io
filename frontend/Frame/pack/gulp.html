一、gulp介绍 #

前端自动化打包构建工具，基于流格式的自动化构建工具。

流格式：需要从源头一步步开始加工，每一步都需要依赖上一步的结果，最后完成一个完整的工作

1.1 依赖环境 #

  • 依赖于node环境进行开发，底层封装的内容就是node里面的读写文件

1.2 作用 #

  • css文件：压缩、转码(自动添加前缀)
  • js文件：压缩、转码(ES6转ES5)
  • html文件：压缩、转码(格式处理)
  • 对静态资源文件和第三方文件都可以进行处理

2.2 创建 gulpfile 文件 #

在项目根目录下创建一个名为 gulpfile.js 的文件

具体解释来自gulp中文官网：

gulpfile 是项目目录下名为 gulpfile.js （或者首字母大写 Gulpfile.js，就像 Makefile 一样命名）的文件，在运行 gulp 命令时会被自动加载。在这个文件中，你经常会看到类似 src()、dest()、series() 或 parallel() 函数之类的 gulp API，除此之外，纯 JavaScript 代码或 Node 模块也会被使用。任何导出（export）的函数都将注册到 gulp 的任务（task）系统中。

三、gulp常用API #

3.1 gulp.task #

创建一个基于流的任务

  • 语法：gulp.task(任务名称,加处理函数)

    gulp.task("test",function(){})

3.2 gulp.src #

找到源文件

  • 语法： gulp.src(路径信息)

gulp.src('./a/b/test.html'):指定特定文件

gulp.src('./a/*.html'):指定目录下特定后缀的文件

gulp.src('./a/b/test.html'):指定特定文件

gulp.src('./a/**):指定目录下所有文件

gulp.src('./a/** / *'):指定目录下和子目录下所有文件

gulp.src('./a/** / *.html'):指定目录下和子目录下特定后缀的文件

3.3 gulp.dest #

把内容放到一个指定目录去

  • 语法： gulp.dest(路径信息)

gulp.dest('./dist'):把接收到的内容放到dist目录下

3.4 gulp.watch #

监控指定目录下的文件，一旦发生变化就执行对应的任务

  • 语法：gulp.watch(路径信息，任务名称)

gulp.watch('./src/ ** / *',function(){})

3.5 gulp.series #

逐步执行多个任务，前一个任务执行完，执行下一个任务，串行

  • 语法：gulp.series(任务1,任务2,任务三,....)

    gulp.series(function(){},function(){})

3.6 gulp.parallel #

并行多个任务

gulp.parallel(function(){},function(){})

3.7 pipe() #

管道函数，所有gulp API都是基于流,接收当前流，进入下一个流过程的管道函数

gulp.src('./a/b/test.html').pipe(压缩任务).pipe(转码任务).pipe(gulp.dest('./dist'))

const gulp = require('gulp')
const cssmin = require('gulp-cssmin')
const rename = require('gulp-rename')
const autoprefixer = require('gulp-autoprefixer')
const sass =  require('gulp-sass')(require('sass'))

//生成soucemaps的，
const sourcemaps = require('gulp-sourcemaps')
const sassHander = function(cb) {
     // 找到对应目录的css文件
     gulp.src('./src/sass/*.scss')
     .pipe(sourcemaps.init())
     //开启sass自己的压缩和错误日志输出
     .pipe(sass({outputStyle: 'compressed'}).on('error', sass.logError))
     //自动添加前缀，兼容老的浏览器
     .pipe(autoprefixer())
     //压缩
     .pipe(cssmin())
     .pipe(rename({
         dirname:'',//清空路径，否则连css所在文件目录也一并放到dist/css下了。我想要的是dist/css下直接放css文件，没有其他目录了。
         suffix:'.min',//后缀
     }))
     //默认情况下，gulpsourcemaps在编译的CSS文件中内联写入源映射。要将它们写入一个单独的文件，请指定相对于文件的路径
     .pipe(sourcemaps.write('../maps'))
     //放到指定目录
     .pipe(gulp.dest('./dist/sass/'))
      //需要回调出去，让gulp捕获改任务的结束
      cb()
}
exports.sassHander = sassHander;

const jsHandler = function (cb) {

}

9.2 创建一个删除任务 #

执行的过程中我们会发现啊，dist文件夹下面存在的之前存在的东西gulp默认都不会删除，如果我们想每次移动之前，删除dist原有的东西，可以下载依赖

shell
pnpm install del -D
1

编写任务：

js
//require() of ES modules is not supported.
const delPromise = import('del');
const cleanHandler = function(cb) {
    //直接删除即可，不需要流
    //参数以数组的形式传递你要删除的文件夹
    delPromise.then(res=>{
        console.log(res)
        res.deleteSync(['dist/'])
        cb()
    })
}
//创建一个默认任务
module.exports.default =gulp.series(cleanHandler,gulp.parallel(cssHandler,cssHandler,jsHandler,htmlHandler,imageHandler,libHandler),done=>{
    console.log('-----------全部执行完毕------------------')
    done();
})
