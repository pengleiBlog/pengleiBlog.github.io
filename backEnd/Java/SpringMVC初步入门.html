public class BaseServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
       //BookServlet?method=add&bookName=test
        String methodStr = req.getParameter("method");
        try {
            Class c = this.getClass();
            //获取add方法
            Method method = c.getDeclaredMethod(methodStr, Book.class);

            //将request中的数据取出来，存到Book对象，传递到add方法
            Class<?>[] parameterTypes = method.getParameterTypes();//获取到add方法的参数类型 Book
            Class type = parameterTypes[0];// type Book 类
            Object o = type.newInstance();// o就表示一个book对象
            /*
             BeanUtils.populate 这个方法会遍历map<key, value>中的key，
              如果bean中有这个属性，就把这个key对应的value值赋给bean的属性。
                常用于接收请求数据时的封装
             */
            BeanUtils.populate(o,req.getParameterMap());

            String url = (String)method.invoke(this,o);//调用add方法

            if(url.indexOf(":")>=0) {
                String[] arr = url.split(":");
                String t = arr[0];//redirect
                String path = arr[1];
                if("redirect".equals(t)) {
                    resp.sendRedirect(path);
                }else if("forward".equals(t)) {
                    req.getRequestDispatcher(path).forward(req,resp);
                }
            }else {
                req.getRequestDispatcher(url).forward(req, resp);
            }

        } catch (NoSuchMethodException e) {
            e.printStackTrace();
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        } catch (InstantiationException e) {
            e.printStackTrace();
        } catch (InvocationTargetException e) {
            e.printStackTrace();
        }

    }
}

二、Spring MVC概述 #

Spring MVC 是由Spring官方提供的基于MVC设计理念的web框架

Spring MVC 是基于Servlet封装的用于实现MVC控制的框架，实现前端和后端的交互

2.1 SpringMVC 优势 #

  • 严格遵循MVC分层思想
  • 采用了松耦合、插件式结构；相对于上面我们自己封装的BaseServlet以及一些其他的MVC框架来说更灵活，更具有扩展性
  • SpringMVC是基于Spring的扩展，提供了一套完善的MVC注解
  • SpringMVC在数据绑定、视图解析都提供了多种处理方式，可灵活配置
  • SpringMVC对RESFful URL设计方式提供了良好的支持。

2.2 SpringMVC本质工作 #

  • 接收并解析请求
  • 处理请求
  • 数据渲染、响应请求

4.2.1 /*和/的区别 #
  • 拦截所有的HTTP请求，包括.jsp的请求处理，都作为控制器的请求路径来处理
  • 拦截所有的HTTP请求，但不包括.jsp的请求处理，都作为控制器的请求路径来处理。但不放行静态资源请求(html/js/css/图片)

五、SpringMVC请求处理流程 #

5.1 请求处理流程 #

SpringMVC通过前端控制器(DispatcherServlet)拦截并处理用户请求

5.2 SpringMVC核心组件 #

  • DispatcherServlet 前端控制器、总控制器

    • 作用：接收请求，协调各组件工作，响应请求

  • HandlerMapping 处理器映射

  • 作用：负责用户请求的URL找到对应的Handler

  • 可配置，SpringMVC提供多个映射器的实现，可根据需要进行配置

  • HandlerAdapter 处理器适配器

    • 作用：按照处理器映射器解析的用户请求的调用链，通过适配器完成Handler的调用

  • Handler 处理器/控制器

    • 由工程师根据业务的需求进行开发

    • 作用：处理请求

  • ModelAndView 视图模型

    • 作用：用户封装处理器返回的数据以及响应的视图

    • ModelAndView = Model +View

  • ViewResolver 视图解析器

    • 作用：对ModelAndView进行解析

    • 可配置，SpringMVC提供多个视图解析器的实现，可根据需要进行配置

  • View视图

  • 作用：完成视图的渲染

5.3 处理器映射器 #

不同的处理映射器对URL的处理方式也不相同，使用对应的处理器映射器之后我们的前端请求规则也需要发生相应的变化

SpringMVC提供的处理器映射器：

  • BeanNameUrlHandlerMapping：根据控制器的name或者id来访问控制器(默认)

9.1 拦截器介绍 #

SpringMVC提供的拦截器类似于Servlet-api中的过滤器，可以对控制器的请求进行拦截实现相关的预处理和后处理。

  • 过滤器
    • 是Servlet规范的一部分。所有的web项目都可以使用过滤器。
    • 过滤器在web.xml配置(可以使用注解)，能够拦截所有web请求。
  • 拦截器
    • 是SpringMVC框架的实现，只有在SpingMVC框架中才能使用
    • 拦截器是在SpringMVC的配置文件进行配置，不会拦截SpringMVC放行的资源(jsp\html\css)

public class MyInterceptor implements HandlerInterceptor {
    //预处理方法
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("--------预处理-----");
        Enumeration<String> keys = request.getParameterNames();
        while (keys.hasMoreElements()) {
            String key = keys.nextElement();
            if("bookId".equals(key)) {
                return  true;
            }
        }
        response.setStatus(400);
        return false;
    }
   //后处理方法
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        modelAndView.addObject("tips","这是通过拦截器传递的后处理添加产生的数据");
        System.out.println("--------后处理-----");
    }
}
