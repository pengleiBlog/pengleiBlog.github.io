二、SpringBoot的简介 #

2.1 概念 #

  • 随着动态语言的流行，Java语言的开发就显得格外笨重：配置繁琐、开发效率低、项目部署变得复杂、集成第三方的技术难度大

  • 在这种情况下,SpringBoot框架就应运而生了。

  • SpringBoot采用了习惯优于配置/约定大于配置的理念快速的搭建项目的开发环境，我们无需或者进行很少的配置就能快速将项目搭建起来。

    更多说明我们可以参考：官网介绍

2.1.1 SpringBoot的特征 #

  • 创建独立的Spirng应用程序

    独立的含义就是能够独立的开发、独立部署、独立运行，因为最终的编译后的结果、第三方的Jar和容器都打成了一个可执行的jar包，使用java -jar 就可以直接运行了

    以前的传统方式是编译成一个WAR包，然后放到Tomcat等容器里，通过启动容器才能运行。

  • 提供"入门"依赖项以简化构建配置，尽可能自动配置Spring和第三方库

    提供很多 starter依赖，提供很多常用组件的默认配置，简化我们的配置。比如：开发web应用，就用spring-boot-starter-web，数据持久化就用spring-boot-starter-data-xxx

  • 提供 product-ready(生产就绪)功能，例如指标、外部检查和外部化配置。

product-ready指应用在编码完成后，能否直接上到生产上去？还需要做哪些工作才能发布到生产，即生产就绪。

2.2 优点 #

  • 能够快速的搭建项目
  • 对主流的开发框架都提供了无配置的集成。(SpingBoot内置了配置)
  • 项目可以独立的运行，无需单独配置Servlet容器(内置了Tomcat)
  • 极大的提高了开发和部署效率
  • 提供运行时的监控系统(日志等)
  • 与云原生有天然的集成

2.3 缺点 #

  • 由于配置都是内置的,报错时定位比较困难
  • 版本迭代速度比较快、有些版本改动还是比较大的。(增加学习成本)

六、Thymeleaf #

Thymeleaf是一种类似于jsp的动态网页技术

6.1 Thymeleaf简介 #

  • JSP必须依赖Tomcat运行，不能直接运行在浏览器中
  • HMTL可以直接运行在浏览器中
  • Thymeleaf是一种既保留了HTML后缀能够直接在浏览器中运行的能力，又实现了JSP显示动态数据的功能
